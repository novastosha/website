Instructions & How-to-play

                

                    
OBJECTIVE: Reduce the data stream size below the Target Cost.


                    
TOOLS:

                    
    
                        
  • 
                            RLE (Run-Length Encoding):
                            Compress sequential identical characters.
    
                            Ex: "AAAA" (4 cost) → "4A" (2 cost).
                        
    • 
                        
  • 
                            PATTERN ($):
                            Define a sequence as a reusable variable.
    
                            Ex: "ABAB" → "$1". Great for repeating complex
                                segments.
                        
    • 
                        
  • 
                            DECONSTRUCT:
                            Break a compressed token back into raw data to re-optimize.
                        
    • 
                    


                    
CONTROLS:

                    

                        Desktop: Click and drag, or Shift+Click to select range.

                        Mobile: Tap Start then Tap End to select a range.
                    

                

                
            

        


        

            
Compression Puzzle

            

                Compression Puzzle is a logic game about structure.
                

                You are given a fixed target string. Your goal is to recreate it exactly using the smallest possible
                description — by grouping patterns, repeating runs, and defining
                    reusable pieces.
                

                

                Can you describe the same data using less information?
